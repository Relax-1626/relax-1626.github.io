[BJDCTF2020]Cookie is so stable

题目来说并不算难，主要是通过题目来总结一些 SSTI 的发现、分别、使用。

题目的主页面、flag 页面和 hint 页面进行请求数据抓包无异常，但是在 hint 页面的响应数据包中发现：

<!-- Why not take a closer look at cookies? -->

hint 页面是没有输出点的，所以应该还有其他位置存在输出点和注入点。来到 flag 页面，输入 {{7*7}} 验证存在 SSTI。对该页面进行刷新，抓请求包，在 cookie 位置存在一个传参 user，确定注入点。

然后就是该考虑：存在 SSTI，那么他到底是什么引擎呢？

服务器模板注入 (SSTI)

服务器模板注入是一种 web 应用安全漏洞，发生在应用程序使用模板引擎渲染用户输入时。攻击者能够通过将恶意代码注入到模板中，然后在服务器执行。

原理

模板引擎：用来动态生成 HTML、邮件和其他文档。模板包含静态部分和动态变量（使用 {{...}}、{%...%} 等语法标记）。

其实模板引擎就是一种将静态模板与动态数据结合生成最终 HTML（或其它格式）文档的工具，它让开发者可以把页面的结构骨架和业务逻辑数据（变量/对象）分离开来，避免了在代码里到处拼接字符串的混乱局面。

打个比方：
模板引擎 = 填空题

  • 写好了“填空模板”：你好，{{ name }}
  • 引擎从数据库拿到数据：{name:"张三"}
  • 填进填空里，页面最终输出：你好，张三

如果没有模板引擎，后端代码会变成：

# 难以维护，容易引发 XSS 漏洞
response = "<h1>欢迎 " + username + "</h1><p>" + content + "</p>"

有了模板引擎后：

<!-- 视图清晰，安全过滤更方便 -->
<h1>欢迎 {{ username }}</h1>
<p>{{ content }}</p>

常见模板引擎 SSTI 风险与利用

Python

Jinja2 (Flask 等)

  • 沙箱：存在，需手动启动 SandboxedEnvironment。默认环境不安全。
  • SSTI：存在
  • 利用方式：通过 Python 对象自省，从基本类型(如字符串、字典)回溯 object 到基类，进而调用 os 或 subprocess 模块。
  • 常见利用函数/技巧：{{7*7}} 用于探测，利用 __class__、__mro__、__subclasses__、__globals__ 等魔术方法。

Mako (Pyramid 等)

  • 沙箱：默认无。
  • SSTI：存在。
  • 利用方式：允许在模板中嵌入纯 Python 代码块（<%...%>），可以执行任意 Python 语句。
  • 常见利用函数/技巧：<% import os %> <% os.system('id') %>

Django Template

  • 沙箱：默认有较强的沙箱。
  • SSTI：较安全。
  • 利用方式：利用难度较高，通常需要配合其他漏洞(如 SQL 注入)或利用模板过滤器的漏洞(主要指滥用)。
  • 常见利用函数/技巧：{{ user.groups.all }} (信息泄露)

PHP

Twig (Symfony 等)

  • 沙箱：有，可以通过配置启用。
  • SSTI：存在。
  • 利用方式：即使开启沙箱，也可以通过注册未定义过滤器回调或滥用内置 filter/map 过滤器来执行 PHP 函数。
  • 常见利用函数/技巧：{{7*7}} 探测，{{_self.env.registerUndefinedFilterCallback("exec")}}{{_self.env.getFilter("id")}} 执行命令

Smarty

  • 沙箱：有安全模式，限制可用函数。
  • SSTI：存在
  • 利用方式：在旧版本中，可直接调用 {php} 标签；在新版中，可通过 {self::getStreamVariable()} 读文件或 {capture} 执行 PHP 代码。
  • 常见利用函数/技巧：{$smarty.version} 探测，{self::getStreamVariable("file:///etc/passwd")} 读文件，{literal}{/literal} 标签利用。

Blade

  • 沙箱：无，但输出默认转义。
  • SSTI：理论上存在。
  • 利用方式：相对困难。通常需要利用 @php 指令或特定场景下未转义的 {!! $var !!} 语法。
  • 常见利用函数/技巧：@php system('id');@endphp

Java

FreeMarker

  • 沙箱：有，但默认不安全。
  • SSTI：存在。
  • 利用方式：通过 new() 内建函数实例化 freemarker.template.utility.Execute 类来执行系统命令。
  • 常见利用函数/技巧：${7*7} 探测，<#assign ex="freemarker.template.utility.Execute"?new()> ${ ex("id") }

Velocity

  • 沙箱：功能较弱。
  • SSTI：存在。
  • 利用方式：通过反射获取 java.lang.Runtime 类来执行命令。
  • 常见利用函数/技巧：#set($runtime = ...) $runtime.exec('id')

Thymeleaf (Spring Boot)

  • 沙箱：Spring Boot 新版增强了表达式限制。
  • SSTI：存在
  • 利用方式：利用 __${...}__ 预处理表达式进行双重求值，并尝试绕过新版 Spring Boot 的 T() 操作符限制。
  • 常见利用函数/技巧：${7*7} 探测，${T(java.lang.Runtime).getRuntime().exec('calc')} 新版可能被禁。

Node.js/JavaScript

Pug (原名 Jade)

  • 沙箱：无
  • SSTI：存在
  • 利用方式：在 - 开头的代码块中可直接编写 JavaScript 代码，访问全局对象 global.process 来执行系统命令。
  • 常见利用函数/技巧：#{7*7} 探测，- global.process.mainModule.require('child_process').execSync('id')

EJS

  • 沙箱：无。
  • SSTI：存在。
  • 利用方式：通过 <%- 等标签注入代码，其 include 指令可能存在路径遍历，结合原型链污染可达 RCE。
  • 常见利用函数/技巧：<%= 7*7 %> 探测，<%- global.process.mainModule.require('child_process').execSync('id') %>

Handlebars

  • 沙箱：有，其“无逻辑”设计本身就是一种安全限制。
  • SSTI：RCE 困难
  • 利用方式：主要利用点在于，若开发者将用户输入传给 compile 函数，可能导致拒绝服务(DoS)或有限的信息泄露。
  • 常见利用函数/技巧：{{#with "s" as |string|}}...{{/with}} 构造复杂数据，实现原型链污染。

Nunjucks

  • 沙箱：有，但可能被绕过。
  • SSTI：存在。
  • 利用方式：通常需要结合原型链污染或利用 eval 等内置过滤器来达到 RCE。
  • 常见利用函数/技巧：{{7*7}} 探测，{{range.constructor("return global.process.mainModule.require('child_process').execSync('id')")()}}

Ruby

ERB

  • 沙箱：默认无。
  • SSTI：存在。
  • 利用方式：<%= %> 标签内可执行任意 Ruby 代码，包括 system 命令。
  • 常见利用函数/技巧：<%= 7*7 %> 探测，<%= system('id') %>

Slim

  • 沙箱：默认无。
  • SSTI：存在。
  • 利用方式：与 ERB 类似，其 - 和 = 标签允许执行任意 Ruby 代码。
  • 常见利用函数/技巧：- system('id') =7*7

Liquid

  • 沙箱：有较强沙箱。
  • SSTI：存在。
  • 利用方式：沙箱严格限制了可执行代码，但设计缺陷可能导致任意文件读取。
  • 常见利用函数/技巧：{{'file:///etc/passwd'|file_excerpt}} 等过滤器利用。

Go

text/template

  • 沙箱：无
  • SSTI：存在。
  • 利用方式：可调用模板上注册的自定义函数，通过 call 调用 Go 标准库中的公共函数。
  • 常见利用函数/技巧：{{printf "%s" "hello"}} 探测，{{.User.Password}} 信息泄露，{{call .User.UpdatePassword "new_pass"}} 函数调用。

html/template

  • 沙箱：有，相比 text/template 更安全。
  • SSTI：存在。
  • 利用方式：受限于安全设计，但若开发者使用 template.HTML 等类型包裹用户输入，可能导致 XSS 或有限的模板注入。
  • 常见的利用函数/技巧：{{.}} 探测
